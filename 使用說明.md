# 00981A ETF 持股變化追蹤系統 - 使用說明

## 功能特點

✅ **自動抓取持股數據** - 每天自動從網站抓取 00981A ETF 的持股明細  
✅ **自動比較變化** - 自動比較前後兩天的持股變化  
✅ **自動儲存檔案** - 每天自動儲存帶日期時間的檔案  
✅ **自動發送通知** - 自動發送持股明細和更新報告到 Telegram Bot  
✅ **排程功能** - 每天 18:00（晚上6點）自動執行
✅ **GitHub Actions 支援** - 可在 GitHub 上自動執行，無需本地電腦開機  

## 安裝步驟

### 1. 安裝 Python 依賴

```bash
pip install -r requirements.txt
```

### 2. 設定 Telegram Bot

1. 在 Telegram 搜尋你的 Bot（使用 Bot Token）
2. 對 Bot 發送任意一則訊息（例如：hi）
3. 執行：`python get_chat_id.py`
4. 把顯示的 Chat ID 填進 `config.py` 的 `TELEGRAM_CHAT_ID`

### 3. 設定 ChromeDriver（可選）

如果使用 Selenium 版本，需要 ChromeDriver：
- **方法1**: `pip install webdriver-manager`（自動下載）
- **方法2**: 手動下載 ChromeDriver 並添加到 PATH
- **方法3**: 將 `chromedriver.exe` 放在項目目錄中

## 使用方法

### 方法一：使用批次檔（推薦）

雙擊 `run.bat`，然後選擇：
- **選項 1**: 立即執行一次（測試用）
- **選項 2**: 啟動排程器（每天6點自動執行）

### 方法二：使用命令列

#### 立即執行一次
```bash
python main.py --now
```

#### 啟動排程器（每天6點自動執行）
```bash
python main.py
```

## 排程設定

### 本地排程器

程式預設每天 **18:00**（晚上6點）自動執行。如需修改時間，請編輯 `main.py` 中的：

```python
schedule.every().day.at("18:00").do(fetch_and_send)
```

例如改為每天 20:00：
```python
schedule.every().day.at("20:00").do(fetch_and_send)
```

### GitHub Actions（推薦）

使用 GitHub Actions 可以完全自動化，不需要本地電腦開機。詳細設定請參考：**[GitHub設定指南.md](GitHub設定指南.md)**

- **執行時間**：每天台灣時間 18:00（晚上6點）
- **時區**：自動轉換為 UTC 時間（台灣時間 18:00 = UTC 10:00）
- **修改時間**：編輯 `.github/workflows/daily-fetch.yml` 中的 cron 設定

## 檔案說明

- `main.py` - 主程式（整合版，支援排程）
- `config.py` - 配置文件（Telegram Bot Token 等）
- `scraper_requests.py` - requests 版本的爬蟲（不需要 ChromeDriver）
- `scraper_selenium.py` - Selenium 版本的爬蟲（更可靠）
- `holdings_data.json` - 最新持股數據（供下次比較用）
- `holdings_data_YYYY-MM-DD_HHMM.json` - 帶日期時間的歷史數據檔案
- `run.bat` - Windows 快速啟動腳本

## 執行流程

1. **每天 18:00（晚上6點）自動執行**
2. **抓取數據** - 先嘗試 requests，失敗則使用 Selenium
3. **比較變化** - 與前一天的數據比較
4. **儲存檔案** - 自動儲存帶日期時間的檔案
5. **發送通知** - 發送兩則訊息到 Telegram：
   - 第一則：今日持股明細
   - 第二則：與前日比較的變化報告

## 注意事項

1. **保持程式運行** - 排程器需要持續運行才能每天自動執行
2. **網絡連接** - 確保電腦有穩定的網絡連接
3. **Chrome 瀏覽器** - 如果使用 Selenium，需要安裝 Chrome 瀏覽器
4. **Telegram Bot** - 確保 Bot Token 和 Chat ID 設定正確

## 故障排除

### 無法抓取數據
1. 檢查網絡連接
2. 確認網站可訪問：https://www.pocket.tw/etf/tw/00981A/fundholding
3. 如果 Selenium 失敗，程式會自動嘗試 requests 版本

### Telegram 發送失敗
1. 確認 Bot Token 正確
2. 先發送一條消息給 Bot，讓程式獲取 chat_id
3. 檢查網絡連接（Telegram API 可能需要代理）

### 排程器不執行
1. 確認程式持續運行中
2. 檢查系統時間是否正確
3. 確認沒有其他程式占用資源

## 技術支援

如有問題，請檢查：
- `config.py` 中的設定是否正確
- Python 版本是否為 3.7+
- 所有依賴套件是否已安裝
