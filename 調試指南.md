執行時會顯示：

### 步驟 4：手動測試 Selenium
如果自動抓取失敗，可以手動測試：

```python
from selenium import webdriver
from selenium.webdriver.chrome.options import Options

options = Options()
options.add_argument('--headless')
driver = webdriver.Chrome(options=options)
driver.get("https://www.pocket.tw/etf/tw/00981A/fundholding")
print(driver.title)
print(driver.page_source[:1000])  # 查看前1000字元
driver.quit()
```

## 常見問題

### 問題 1：API 請求失敗
**症狀**：顯示 "API 回應非 200" 或 "API 請求錯誤"

**可能原因**：
- 網站結構改變
- API 端點變更
- 需要認證或特殊 headers

**解決方案**：
1. 檢查網站是否有更新
2. 查看瀏覽器開發者工具的 Network 標籤，找到實際的 API 端點
3. 更新 `scraper_selenium.py` 中的 API URL 列表

### 問題 2：Selenium 超時
**症狀**：顯示 "Selenium 逾時" 或 "頁面載入逾時"

**可能原因**：
- 網頁載入太慢
- 需要等待 JavaScript 執行
- 網路連接不穩定

**解決方案**：
1. 檢查網路連接速度
2. 增加超時時間（已改為 60 秒）
3. 嘗試不使用 headless 模式查看實際情況

### 問題 3：無法解析數據
**症狀**：顯示 "無法從網頁中提取持股數據"

**可能原因**：
- 網站 HTML 結構改變
- 數據格式改變
- 需要特殊處理

**解決方案**：
1. 查看詳細日誌，確認找到的元素
2. 手動訪問網站，檢查實際的 HTML 結構
3. 更新解析邏輯以適應新的結構

## 手動檢查網站結構

1. 打開瀏覽器，訪問：https://www.pocket.tw/etf/tw/00981A/fundholding
2. 按 F12 打開開發者工具
3. 查看 Network 標籤，找到實際的 API 請求
4. 查看 Elements 標籤，檢查持股數據的 HTML 結構
5. 查看 Console 標籤，檢查是否有 JavaScript 錯誤

## 臨時解決方案

如果自動抓取持續失敗，可以：
1. 手動訪問網站
2. 複製持股數據
3. 手動更新 `holdings_data.json`

格式範例：
```json
{
  "date": "2/6",
  "holdings": [
    {"code": "2330", "name": "台積電", "shares": 1500},
    {"code": "2454", "name": "聯發科", "shares": 800}
  ]
}
```
