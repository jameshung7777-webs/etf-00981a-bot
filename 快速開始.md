# 00981A ETF 自動追蹤系統 - 快速開始

## 三步驟開始使用

### 1. 安裝依賴
```bash
pip install -r requirements.txt
```

### 2. 設定 Telegram Bot
1. 在 Telegram 對你的 Bot 發送一則訊息
2. 執行：`python get_chat_id.py`
3. 把顯示的 Chat ID 填進 `config.py`

### 3. 啟動程式

**方法一：使用批次檔（最簡單）**
- 雙擊 `run.bat`
- 選擇「2. 啟動排程器」

**方法二：使用命令列**
```bash
python main.py
```

## 功能說明

✅ **每天 18:00（晚上6點）自動執行**
- 自動抓取持股數據
- 自動比較變化
- 自動儲存檔案（帶日期時間）
- 自動發送兩則訊息到 Telegram：
  1. 今日持股明細
  2. 與前日比較的變化報告

## 兩種執行方式

### 方式一：GitHub Actions（推薦）⭐
- **優點**：不需要本地電腦開機，完全自動化
- **設定**：參考 [GitHub設定指南.md](GitHub設定指南.md)
- **執行時間**：每天台灣時間 18:00 自動執行

### 方式二：本地排程器
- **優點**：可以立即看到執行結果
- **缺點**：需要電腦持續開機
- **執行時間**：每天 18:00（可修改）

## 立即測試

如果想立即測試（不等待排程）：
```bash
python main.py --now
```

## 注意事項

⚠️ **排程器需要持續運行**
- 程式必須保持運行狀態
- 可以最小化視窗，但不要關閉
- 按 Ctrl+C 可停止程式

## 檔案說明

- `holdings_data.json` - 最新數據（供比較用）
- `holdings_data_YYYY-MM-DD_HHMM.json` - 歷史數據檔案（每天自動產生）

## 修改執行時間

### GitHub Actions
編輯 `.github/workflows/daily-fetch.yml` 中的 cron 設定

### 本地排程器
編輯 `main.py` 第 188 行：
```python
schedule.every().day.at("18:00").do(fetch_and_send)
```

例如改為 20:00：
```python
schedule.every().day.at("20:00").do(fetch_and_send)
```

## 問題排除

**無法抓取數據？**
- 檢查網絡連接
- 程式會自動嘗試兩種方法（requests 和 Selenium）

**Telegram 發送失敗？**
- 確認 Bot Token 和 Chat ID 設定正確
- 先對 Bot 發送訊息，再執行 `get_chat_id.py`

**排程不執行？**
- 確認程式持續運行中
- 檢查系統時間是否正確
